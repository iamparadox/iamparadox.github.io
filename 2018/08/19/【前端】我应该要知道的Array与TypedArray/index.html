【前端】我应该要知道的Array与TypedArray

iamparadox 发表于 2018-08-19 |
字数统计: 3,554 | 阅读时长 ≈ 14

       Array类型配合着字符串是实质生产过程中最常用的部分，许多涉及到排序、比较、都是通过类型转换成数组类型通过其丰富的 API 进行处理的。在 JS 的内置类型中，数组和function都隶属于object分支之一,故此在使用typeof时要格外注意返回结果，必要时通过 instanceof 函数来区分结果集。

Array

属性 来源于 MDN

Array.prototype 返回了数组的原型。

Array.prototype.constructor 用于创造对象的原型对象的特定的函数。
Array.prototype.length 用于创造对象的原型对象的特定的函数。
Array.length 返回了数组的长度。
Array[Symbol.unscopables] 防止某些数组方法被添加到 with 语句的作用域内。

Api 来源于 MDN

  • Array.from()

    Array.from() 方法从一个类似数组或可迭代对象中创建一个新的数组实例。

  • Array.isArray()

    Array.isArray()
    方法用于确定传递的值是否是一个 Array。返回布尔值

  • Array.of()

    Array.of()
    方法创建一个具有可变数量参数的新数组实例，而不考虑参数的数量或类型。返回一个新数组。
    Array.of(7)和 Array(7)的区别:
    前者定义一个空数组并push入一数字型的7,后者定义一个长度为7的空白单元:[ , , , , , , ]

  • Array.prototype.concat()

    Array.prototype.concat(value1[, value2[, …[, valueN]]])
    方法用于合并两个或多个数组。此方法不会更改现有数组，而是返回一个新数组。

  • concat返回一个浅拷贝，也就是说修改引用类型会影响原型链中的引用。

  • 可以通过设置Symbol.unscopable = false; 来让其不展开合并。

  • Array.prototype.copyWithin()

    Array.prototype.copyWithin(target[, start[, end]]) 参数1:复制序列到该位置。默认0，负数从末尾开始计算。参数2:起始位置；参数3：结束位置；
    方法浅复制数组的一部分到同一数组中的另一个位置，并返回它，而不修改其大小。
    将会改变原数组！！！

  • Array.prototype.entries()

    Array.prototype.entries()
    方法返回一个新的Array Iterator对象，该对象包含数组中每个索引的键/值对。

  • Array.prototype.every()

    Array.prototype.every(callback[, thisArg]) callback 被调用时传入三个参数：元素值，元素的索引，原数组。
    方法测试数组的所有元素是否都通过了指定函数的测试。

  • Array.prototype.fill()

    Array.prototype.fill(value[, start[, end]]) 参数1：用来填充数组元素的值。参数2：起始索引，默认值为0。参数3：终止索引，默认值为0。
    方法用一个固定值填充一个数组中从起始索引到终止索引内的全部元素。不包括终止索引。

  • Array.prototype.filter()

    Array.prototype.filter(callback(element[, index[, array]])[, thisArg]) callback 被调用时传入三个参数：元素值，元素的索引，被遍历的数组。
    方法创建一个新数组, 其包含通过所提供函数实现的测试的所有元素。不会改变原数组，它返回过滤后的新数组。

我一般用forEach做，下次可以试试这个。

  • Array.prototype.find()

    Array.prototype.find(callback[, thisArg]) callback 被调用时传入三个参数：元素值，元素的索引，被遍历的数组。
    方法返回数组中满足提供的测试函数的第一个元素的值。否则返回 undefined。不会改变原数组，它返回过滤后的新数组。

这个很常用了。

  • Array.prototype.findIndex()

    Array.prototype.findIndex(callback[, thisArg]) callback 被调用时传入三个参数：元素值，元素的索引，被遍历的数组。
    方法返回数组中满足提供的测试函数的第一个元素的索引。否则返回-1。不会改变原数组，它返回过滤后的新数组。

这个感觉不是很有必要。find()方法就能得到索引，可以输出出来。

  • Array.prototype.forEach()

    Array.prototype.forEach(callback(currentValue, index, array){}, this);
    方法对数组的每个元素执行一次提供的函数。不会改变原数组，它返回过滤后的新数组。
    不可以中途退出遍历：forEach() 为每个数组元素执行callback函数；不像map() 或者reduce() ，它总是返回 undefined值，并且不可链式调用。典型用例是在一个链的最后执行副作用。

  • Array.prototype.includes()

    Array.prototype.includes(searchElement, fromIndex); 参数一：需要查找的元素值。参数二：从该索引处开始查找。默认0，可以为负；返回一个布尔值
    方法对数组的每个元素执行一次提供的函数。

  • Array.prototype.indexOf()

    Array.prototype.indexOf(searchElement[, fromIndex = 0]) 参数1:被查找的;参数2：开始查找的索引
    方法返回在数组中可以找到一个给定元素的第一个索引，如果不存在，则返回-1。

  • Array.prototype.join()

    str = arr.join() // 默认为 “,” separator的话则所有元素之间都没有任何字符。 返回一个字符串
    方法将一个数组（或一个类数组对象）的所有元素连接成一个字符串并返回这个字符串。
    这个方法带隐式类型转换，如果是null 或者 undefined,则会转化成空字符串。

  • Array.prototype.keys()

    Array.prototype.keys()
    方法返回一个包含数组中每个索引键的Array Iterator对象。
    暂时不知道是什么。先留个印象

  • Array.prototype.map()

    var new_array = arr.map(function callback(currentValue[, index[, array]]) { return XXX }[, thisArg])
    方法创建一个新数组，其结果是该数组中的每个元素都调用一个提供的函数后返回的结果。
    map()方法必须要 return出来，否则容易死循环

  • Array.prototype.pop()

    Array.prototype.pop()
    方法从数组中删除最后一个元素，并返回该元素的值。此方法更改数组的长度。
    返回从数组中删除的元素(当数组为空时返回undefined)。

  • Array.prototype.unshift()

    Array.prototype.unshift();
    方法将一个或多个元素添加到数组的开头，并返回新数组的长度。

  • Array.prototype.shift()

    Array.prototype.shift()
    方法从数组中删除第一个元素，并返回该元素的值。此方法更改数组的长度。

  • Array.prototype.push()

    Array.prototype.push(element1, …, elementN)
    方法将一个或多个元素添加到数组的末尾，并返回新数组的长度。

使用技巧 合并数组

  • Array.prototype.reduce()

    Array.prototype.reduce(callback[, initialValue]) initialValue：初值

callback接受四个参数：

  • accumulator 累加器 : 每次结束后上一次结果更新成这个值，有初始值则为初始值，默认是数组[0]
  • currentValue
  • currentIndex
  • array

如果传入初值了，则accumulator取初值，currentValue取数组第一个值；否则分别取arr[0],arr[1];
有初值更安全，MDN 讲到处理时可以先用map(el => el.x).reduce(callback,-Infinity)空的数组处理不会报错。

方法对累加器和数组中的每个元素（从左到右）应用一个函数，将其减少为单个值。
返回效果和’+’一样，可能是字符串拼接，可能是数字累加。
每一次循环必须有 return,否则accumulator无法获取到值。

  • Array.prototype.reverse()

    Array.prototype.reverse()
    方法将数组中元素的位置颠倒。这个在处理字符串反向时非常常用。

  • Array.prototype.slice()

    Array.prototype.slice(begin, end); 参数1：开始索引，默认0；参数2：结束索引，默认0；参数2指向的不会被输出!!!!
    方法返回一个从开始到结束（不包括结束）选择的数组的一部分浅拷贝到一个新数组对象。且原始数组不会被修改。
    返回一个新数组哦亲。这是浅拷贝哦，其中的子项如果带引用，这个引用更改还是会更改，但是新数组中私有的增删不会改变调用这个方法的原数组。

  • Array.prototype.some()

    Array.prototype.some(callback[, thisArg]) callback接收三个参数：currentValue，index，array(被调用的数组)
    方法测试数组中的某些元素是否通过由提供的函数实现的测试
    实测会全部遍历一遍。和forEach 区别是，有真则立刻可以退出当前的循环。

  • Array.prototype.sort()

    Array.prototype.sort([compareFunction]) compareFunction：用来指定按某种顺序进行排列的函数。如果省略，元素按照转换为的字符串的各个字符的Unicode位点进行排序。
    方法用原地算法对数组的元素进行排序，并返回数组。排序不一定是稳定的。默认排序顺序是根据字符串Unicode码点。
    此方法会改变原数组，直接输出排序后的原数组，不是复制的。

这个排序不一定正确，我要深入自己测几个demo，在此留个坑。

  • Array.prototype.splice()

    Array.prototype.splice(start[, deleteCount[, item1[, item2[, …]]]]) 参数1：开始位置； 参数2：移除位置。参数3：要被添加的元素。
    方法通过删除现有元素和/或添加新元素来更改一个数组的内容
    由被删除的元素组成的一个数组。如果只删除了一个元素，则返回只包含一个元素的数组。如果没有删除元素，则返回空数组。
    如果没有第三个参数，就是纯删除，有了第三个参数，就是先删除再 PUSH。

和slice 的区别：slice是不会改变原始数组的，这个是会的！！！

  • Array.prototype.toLocaleString()

    Array.prototype.toLocaleString([locales[,options]]);
    方法返回一个字符串表示数组中的元素。数组中的元素将使用各自的 toLocaleString 方法转成字符串，这些字符串将使用一个特定语言环境的字符串（例如一个逗号 “,”）隔开。

  • Array.prototype.toSource()

    Array.prototype.toSource();
    方法返回一个字符串,代表该数组的源代码。不建议使用

    Array.prototype.toString();

TypedArray

语法：

new TypedArray(); // ES2017中新增
new TypedArray(length);
new TypedArray(typedArray);
new TypedArray(object);
new TypedArray(buffer [, byteOffset [, length]]);

TypedArray()指的是以下的其中之一：

Int8Array();
Uint8Array();
Uint8ClampedArray();
Int16Array();
Uint16Array();

Uint32Array();
Float32Array();
Float64Array();

属性 来源于 MDN

TypedArray.prototype 返回了类数组的原型。

TypedArray.prototype.constructor 返回创建实例原型的构造函数.这是相应的typed array type的默认的构造函数.
TypedArray.prototype.length 【只读】返回在类型化数组中的元素的数量.创建时已被固化,因此是只读的.
TypedArray.prototype.buffer 【只读】返回被格式化数组引用的ArrayBuffer. 创建时已被固化,因此是只读的.
TypedArray.prototype.byteLength 【只读】返回从ArrayBuffer读取的字节长度. 创建时已被固化,因此是只读的.
TypedArray.prototype.byteOffset 【只读】返回从ArrayBuffer读取时的字节偏移量.创建时已被固化,因此是只读的.
TypedArray.length 返回了类数组的长度。
TypedArray.BYTES_PER_ELEMENT 返回不同类型的数组对象的元素大小的数字值。
TypedArray.name 返回构造器的名称,例如”Int8Array”。
