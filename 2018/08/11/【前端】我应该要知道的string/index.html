【前端】我应该要知道的string

iamparadox 发表于 2018-08-11 |
字数统计: 2,675 | 阅读时长 ≈ 10

       String类型作为JS的基本类型之一,是平日里构成代码结构的重要组成之一,在使用过程中,常常作为显式或者隐式的被转换类型,通过原生的API进行一些别致的处理,例如字符串反转。它被存储于栈内存中,这导致着我们可以直接操作在变量中的值,也就是按值访问。但是需要注意的是字符串的成员函数并不能更改原始值,往往是创建并返回一个新字符串,这在下面的API中可以看到。
首先说一下属性。

属性 部分来源于 MDN

String.prototype 返回了字符串的原型。

String.prototype.constructor 用于创造对象的原型对象的特定的函数。
String.prototype.length 用于创造对象的原型对象的特定的函数。

String.length 返回了字符串的长度。

       万物皆是对象。String类型实质上也是一个被定义好的String对象,在这个对象里存有两个属性,__proto__和 length;在此需要注意,上述的MDN中的JS的关键词prototype是函数中使用的,而引申到实际存储中的关键词Key实质上是__proto__。并且,__proto__中可以看到当前对象中所有的内部方法,如果需要新增new String().__proto__.CustomFun = function(){},而使用prototype的话仅需String.prototype.CustomFun=function(){},这个可以打印一下一看究竟。

       上面这张图分别打印了new String() 和 String(),根据W3D的说法,当 String() 和运算符 new 一起作为构造函数使用时,它返回一个新创建的 String 对象,存放的是字符串或字符串表示。当不用 new 运算符调用 String() 时,它只把内容转换成原始的字符串,并返回转换后的值。 在此想说,由于JS是词法作用域,编译前已经将定义初始时一些声明走完,故此在使用__proto__和prototype时,如果使用的了关键词New的构造函数(JS中的New并非面向对象的继承,而是通过apply函数copy了一份)所生成的字符串对象里的方法也会被提升。所以String的实例都继承自String.prototype。任何String.prototype上的改变都会影响到所有的String实例。你永远不知道你的猪队友会干什么,因此最好的办法就是谨慎(严禁)覆盖！！！

       上图中可以看到一些被继承的原生Api,如果我们新增了方法,它们也可以显示在__proto__里。知道这点对你驾驭一个第三方的前端库去面对一些奇奇怪怪的需求很有帮助。现在我们来背诵一下原生Api。

Api 部分来源于 MDN

  • String.prototype.anchor()

str.anchor(name) 方法创建一个 <a> HTML 锚元素,被用作超文本靶标（hypertext target） 参数:一个字符串，表示被创建的标签的 name 属性。

  • String.prototype.charAt()

str.charAt(index) 方法从一个字符串中返回指定的字符。参数: (0~length-1) 如果没有提供索引，charAt() 将使用0。

也可以在字符串中使用str[index]

  • String.prototype.charCodeAt()

str.charCodeAt(index) 参数: 一个大于等于 0，小于字符串长度的整数。如果不是一个数值，则默认为 0。返回值是一表示给定索引处（String中index索引处）字符的 UTF-16 代码单元值的数字；如果索引超出范围，则返回 NaN

区分大小写，因为得到的是Unicode编码。

  • String.prototype.concat()

str.concat(string2, string3[, …, stringN]) 参数: string2…stringN 和原字符串连接的多个字符串。

不影响原字符串，生成新字符串返回。性能上比运算符差。

  • String.prototype.endsWith()

str.endsWith(searchString [, position]); 参数1:被搜索的字符串；参数2:搜索的字符串的长度。区分大小写！！！返回 True or False
一个字符串是否在另一个字符串的末尾，这个方法是大小写敏感的。

兼容低版本写法

  • String.prototype.includes()

str.includes(searchString[, position]) 参数1:被搜索的字符串；参数2:搜索开始的位置 默认为0 区分大小写。返回 True or False

方法用于判断一个字符串是否包含在另一个字符串中，根据情况返回 true 或 false。这和indexOf的区别在于，includes()方法直接返回true or false,indexOf 返回 -1 0 ……位置。一个是布尔型 一个是字符型。兼容性比indexOf差很多，不支持IE。

  • String.prototype.indexOf()

str.indexOf(searchValue[, fromIndex]) 参数1:被搜索的字符串；参数2:搜索开始的位置 默认为0 区分大小写。没有返回-1，有返回索引

方法返回调用String对象中第一次出现的指定值的索引，开始在查找位置进行搜索,第二个参数默认是0。如果未找到该值，则返回-1。

  • String.prototype.lastIndexOf()

str.lastIndexOf(searchValue[, fromIndex]) 参数1:被搜索的字符串；参数2:搜索开始的位置 默认为0 区分大小写。没有返回-1，有返回索引

方法返回指定值在调用该方法的字符串中最后出现的位置，如果没找到则返回 -1。从该字符串的后面向前查找，从 fromIndex 处开始。

  • String.prototype.repeat()

str.repeat(count) 参数count:介于0和正无穷大之间的整数 : [0, +∞) 。表示在新构造的字符串中重复了多少遍原字符串。参数会自动向下取整，必须大于0。不支持 IE

方构造并返回一个新字符串，该字符串包含被连接在一起的指定数量的字符串的副本。

  • String.prototype.replace()

str.replace(regexp|substr, newSubStr|function)
参数1regexp:正则对象或者字面量 || 参数1substr:要被替换的字符串

参数2newSubStr预备替换的字符串 || 参数2拼接要替换字符串的函数

方法返回一个由替换值替换一些或所有匹配的模式后的新字符串。模式可以是一个字符串或者一个正则表达式, 替换值可以是一个字符串或者一个每次匹配都要调用的函数。这个真的很常用！！

  • String.prototype.search()

str.search(regexp) 参数：一个正则表达式对象,传入值会隐式转换成正则表达式对象
方法执行正则表达式和 String对象之间的一个搜索匹配。返回匹配索引，否则-1.这点和indexOf()的返回一致。

  • String.prototype.slice()

str.slice(beginSlice[, endSlice]) 参数1：从该索引开始提取，可以为负数，以0为基准。参数2：从该索引结束提取，以0为基准。如果省略默认为最后。
方法提取一个字符串的一部分，并返回一新的字符串。

  • String.prototype.split()

str.split([separator[, limit]]) 参数1：指定拆分时参照项。参数2：数字型，限定数量。
方法使用指定的分隔符字符串将一个String对象分割成字符串数组，以将字符串分隔为子字符串，以确定每个拆分的位置。

这个在处理关键词检索高亮的时候用过。类似的需求还有很多，在前端处理数据时，可以说是非常常用的方法了。输出结果是一个数组，也可以看成类型转换的方法之一了。

  • String.prototype.startsWith()

str.startsWith(searchString [, position]); 参数1：被参照的字符串。参数2：起始索引。
方法用来判断当前字符串是否是以另外一个给定的子字符串“开头”的，根据判断结果返回 true 或 false。

不支持 IE！！和 search()类似作用，但是search()支持IE,且参数传入后的类型也返回结果也不一致。

  • String.prototype.substr()

str.substr(start[, length]); 参数1：开始提取的位置 以0为基准。参数2：提取的字符数量长度，为0则返回空字符串。
方法返回一个字符串中从指定位置开始到指定字符数的字符。

不支持 IE！！和 search()类似作用，但是search()支持IE,且参数传入后的类型也返回结果也不一致。

  • String.prototype.substring()

str.substring(indexStart[, indexEnd]); 参数1：开始提取的位置 以0为基准。参数2：索引结束的。
方法返回一个字符串在开始索引到结束索引之间的一个子集, 或从开始索引直到字符串的末尾的一个子集。

包含给定字符串的指定部分的新字符串。兼容性非常好！！

  • String.prototype.toLocaleLowerCase()

str.toLocaleLowerCase();
方法根据任何特定于语言环境的案例映射，返回调用字符串值转换为小写的值。

  • String.prototype.toLocaleUpperCase()

str.toLocaleUpperCase();
使用本地化（locale-specific）的大小写映射规则将输入的字符串转化成大写形式并返回结果字符串。

  • String.prototype.toLowerCase()

str.toLowerCase();
会将调用该方法的字符串值转为小写形式，并返回。

  • String.prototype.toString()

str.toString();
方法返回指定对象的字符串形式。

  • String.prototype.toUpperCase()

str.toUpperCase();
方法将调用该方法的字符串值转换为大写形式，并返回。

  • String.prototype.trim()

str.trim();
方法会从一个字符串的两端删除空白字符。在这个上下文中的空白字符是所有的空白字符 (space, tab, no-break space 等) 以及所有行终止符字符（如 LF，CR）。

Jquery下的$.trim()

解析：首先尝试使用字符串原生的trim方法（非IE支持），若不支持，使用String.prototype.trim.call(“\uFEFF\xA0”)，若上述两个方法都不支持，使用自定义的正则replace()方法，清空两边的空格或特殊字符。

\uFEFF：某些软件，在保存一个以UTF-8编码的文件时，会在文件开始的地方插入三个不可见的字符（0xEF 0xBB 0xBF，即BOM），转码后是\uFEFF，因此在读取时需要自己去掉这些字符。

\xA0：HTML中常见的 。

  • String.prototype.valueOf()

str.valueOf();
方法返回一个String对象的原始值。
