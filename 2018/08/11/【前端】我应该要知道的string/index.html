【前端 】我应该要知道的string

iamparadox 发表于 2018-08-11 |
字数统计: 1,143 | 阅读时长 ≈ 4

       String类型作为JS的基本类型之一,是平日里构成代码结构的重要组成之一,在使用过程中,常常作为显式或者隐式的被转换类型,通过原生的API进行一些别致的处理,例如字符串反转。它被存储于栈内存中,这导致着我们可以直接操作在变量中的值,也就是按值访问。但是需要注意的是字符串的成员函数并不能更改原始值,往往是创建并返回一个新字符串,这在下面的API中可以看到。
首先说一下属性。

属性 部分来源于 MDN

String.prototype 返回了字符串的原型。

String.prototype.constructor 用于创造对象的原型对象的特定的函数。
String.prototype.length 用于创造对象的原型对象的特定的函数。

String.length 返回了字符串的长度。

       万物皆是对象。String类型实质上也是一个被定义好的String对象,在这个对象里存有两个属性,__proto__和 length;在此需要注意,上述的MDN中的JS的关键词prototype是函数中使用的,而引申到实际存储中的关键词Key实质上是__proto__。并且,__proto__中可以看到当前对象中所有的内部方法,如果需要新增new String().__proto__.CustomFun = function(){},而使用prototype的话仅需String.prototype.CustomFun=function(){},这个可以打印一下一看究竟。

       上面这张图分别打印了new String() 和 String(),根据W3D的说法,当 String() 和运算符 new 一起作为构造函数使用时,它返回一个新创建的 String 对象,存放的是字符串或字符串表示。当不用 new 运算符调用 String() 时,它只把内容转换成原始的字符串,并返回转换后的值。 在此想说,由于JS是词法作用域,编译前已经将定义初始时一些声明走完,故此在使用__proto__和prototype时,如果使用的了关键词New的构造函数(JS中的New并非面向对象的继承,而是通过apply函数copy了一份)所生成的字符串对象里的方法也会被提升。所以String的实例都继承自String.prototype。任何String.prototype上的改变都会影响到所有的String实例。你永远不知道你的猪队友会干什么,因此最好的办法就是谨慎(严禁)覆盖！！！

       上图中可以看到一些被继承的原生Api,如果我们新增了方法,它们也可以显示在__proto__里。知道这点对你驾驭一个第三方的前端库去面对一些奇奇怪怪的需求很有帮助。现在我们来背诵一下原生Api。

Api 部分来源于 MDN

  • String.prototype.anchor()

str.anchor(name) 方法创建一个 <a> HTML 锚元素,被用作超文本靶标（hypertext target） 参数:一个字符串，表示被创建的标签的 name 属性。

  • String.prototype.charAt()

str.charAt(index) 方法从一个字符串中返回指定的字符。参数: (0~length-1) 如果没有提供索引，charAt() 将使用0。

也可以在字符串中使用str[index]

  • String.prototype.charCodeAt()

str.charCodeAt(index) 参数: 一个大于等于 0，小于字符串长度的整数。如果不是一个数值，则默认为 0。返回值是一表示给定索引处（String中index索引处）字符的 UTF-16 代码单元值的数字；如果索引超出范围，则返回 NaN

区分大小写，因为得到的是Unicode编码。

  • String.prototype.concat()

str.concat(string2, string3[, …, stringN]) 参数: string2…stringN 和原字符串连接的多个字符串。

不影响原字符串，生成新字符串返回。性能上比运算符差。

  • String.prototype.endsWith()

str.endsWith(searchString [, position]); 参数1:被搜索的字符串；参数2:搜索的字符串的长度。区分大小写！！！返回 True or False

一个字符串是否在另一个字符串的末尾，这个方法是大小写敏感的。
