typeof

说完内置类型，回过头看看前面的题目:

这里要特别说一下:

  • null被判断成object隶属一个bug;
  • function 并不是一个新的内置类型,在规范中,隶属于object,通过object[[Call]]调用。
  • Array也是隶属于object的一个子类型。

可以看到typeof()方法只能辨别内置类型,但是并不精确。

null 与 undefined 的区别

  • null 曾赋过值，但是目前没有值 或者 内存中不存在改值; 它是关键词。
  • undefined 定义了但是没有赋值。undefined在非严格模式下可以被重新赋值,因为它是标识符。

它们都没有对应的构造形式，只有字面量。
