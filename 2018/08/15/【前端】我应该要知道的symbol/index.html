【前端】我应该要知道的symbol

iamparadox 发表于 2018-08-15 |
字数统计: 1,702 | 阅读时长 ≈ 7

       Symbol类型是ES6标准中新增的基本类型,拥有许多静态属性和静态方法。它的静态属性会暴露几个内建的成员对象；它的静态方法会暴露全局的symbol注册，且类似于内建对象类，但它不是一个构造器，不支持语法：”new Symbol()”。typeof可以输出”symbol”,用它甄别是首选。与其他类型不同的是，Symbol类型没有字面量形式。从定义看，是一个典型的单例模式。

语法

Symbol([description]) 参数:传入一个字符串，相当于描述你为什么要创建这个Symbol类型而已。
注意:传入的description在toString()中只用作这个符号的描述，实际产生的内部 Name 值仍然无法获取。

Symbol.asyncIterator 一个返回对象默认的异步迭代器的方法。使用 for await of。
Symbol正则表达式:

  • Symbol.match 一个用于对字符串进行匹配的方法，也用于确定一个对象是否可以作为正则表达式使用。使用 String.prototype.match()。
  • Symbol.replace 一个替换匹配字符串的子串的方法. 使用 String.prototype.replace()。
  • Symbol.search 一个返回一个字符串中与正则表达式相匹配的索引的方法。使用String.prototype.search()。
  • Symbol.split 一个在匹配正则表达式的索引处拆分一个字符串的方法.。使用 String.prototype.split()。
  • Symbol.hasInstance 一个确定一个构造器对象识别的对象是否为它的实例的方法。使用 instanceof。
  • Symbol.isConcatSpreadable 一个布尔值，表明一个对象是否应该flattened为它的数组元素。使用Array.prototype.concat()。 用来指示如果将其传给一个数组的 concat()是否将其展开。
  • Symbol.unscopables 可以被定义为任何对象的对象属性(Symbol.unscopables),用来指示使用 with语句时哪些属性可以或不可以被暴露。 with()函数一般不推荐使用。
  • Symbol.species 一个用于创建派生对象的构造器函数。
  • Symbol.toPrimitive 一个将对象转化为基本数据类型的方法。
    在 ES6以前很难控制类型转换时的toPrimitive,但是在ES6中在任意对象值上作为属性的符号@@toPrimitivesymbol都可以通过一个指定方法来制定这个toPrimitive强制转换。 更多了解请看这里:【前端】我应该要知道的类型转换们
  • Symbol.toStringTag 用于对象的默认描述的字符串值。使用Object.prototype.toString()。 ES6中: 原型(或实力本身)的@@toStringTag符号指定了在[object ___]字符串化时使用的字符串值。
    TIPS:
    @@hasInstance 符号实在构造器函数上的一个方法，接受实例对象值，通过返回 true or false 来指示这个值是否可以被认为是一个实例。

静态方法

Symbol.for(key) 使用给定的key搜索现有的symbol，如果找到则返回该symbol。否则将使用给定的key在全局symbol注册表中创建一个新的symbol。
Symbol.keyFor(sym) 从全局symbol注册表中，为给定的symbol检索一个共享的symbol key。

作为对象属性中的符号
