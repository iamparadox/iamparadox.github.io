引用

在使用过程中,它被存储于堆内存中,变量赋值即为引用,引用啊兄弟们，一处更改就会影响全部引用此处的地方，由此引申了深拷贝和浅拷贝。

深拷贝 : 设置一个新内存空间去复制。

浅拷贝 : 涉及到了对原型的引用,一朝更改朝不保夕。

Vue中watch的deep模式类似深拷贝，所以对于 Array 和 Object 监听的时候需要使用这个模式，并且不能监听到相应Value的变化，需要配合计算属性。

属性描述符

除了 value 属性外还包含默认的三种属性值：

  • writeable 可读 writeable:false相当于定义一个空的setter,调用setter时报错TypeError。

  • enumerable 可枚举 如果为false 不会出现在对象的属性枚举中，但是仍然可以访问该属性。

  • configurable 可配置 默认属性中configurable为false时可以将writeable的True的可以被配置成，但是false不能被配置成True。不可配置时不能修改 Value 值。

我们可以使用defineProperty()方法重置以上属性。

API 部分来源 MDN

  • Object.prototype.assign()

    Object.assign(target, …sources); 浅拷贝！！！
    方法用于将所有可枚举属性的值从一个或多个源对象复制到目标对象。它将返回目标对象。
    只会拷贝源对象自身的并且可枚举的属性到目标对象。该方法使用源对象的[[Get]]和目标对象的[[Set]]，所以它会调用相关 getter 和 setter。因此，它分配属性，而不仅仅是复制或定义新的属性。如果合并源包含getter，这可能使其不适合将新属性合并到原型中。为了将属性定义（包括其可枚举性）复制到原型，应使用Object.getOwnPropertyDescriptor()和Object.defineProperty() 。

注意，Object.assign 不会跳过那些值为 null 或 undefined 的源对象。

  • Object.prototype.create()

    Object.create(proto, [propertiesObject]);
    方法创建一个新对象，使用现有的对象来提供新创建的对象的proto。
    这个属性我在面试小伙伴的时候经常问。

Object.create()和 new Object()的区别:

  • Object.create()接收两个参数 param1:object; param2:propertiesObject可选，该参数会生成新创建对象的属性与值，其中属性名称将是新创建对象的属性名称，值将是新创建对象的属性描述符，该参数不能是 undefined,且必须拥有可枚举属性。
